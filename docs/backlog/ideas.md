## Текстові поля — валідація за типами

**Дата:** 2025-02-23

**Суть:** Додати до TextInput і TextareaField: `required`, `error`, `pattern`. Валідація за типами даних.

### Текстовий рядок (TextInput)

- **Призначення:** Короткий текст, зазвичай одне конкретне значення (email, телефон, VIN, посилання).
- **Валідація за типом:** Кожне поле має один тип — email, url, phone, slug, pattern (regex). Валідація відповідає обраному типу.
- **Елементи:** required, error, pattern (або тип: email, url, phone тощо).

### Текстове поле (TextareaField)

- **Призначення:** Багато тексту (опис, примітки). Може містити що завгодно: номери, пошти, JSON, вільний текст.
- **Валідація:** Тільки загальна — minLength, maxLength, minRows, maxRows, required. Без тип-специфічної валідації (email/url/phone), бо в одному полі можуть бути різні формати одночасно.
- **Елементи:** required, error. Без pattern за типом.

**Статус:** Впроваджено 2025-02-23.
