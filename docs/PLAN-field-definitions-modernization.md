# План модернізації полів (Field Definitions)

**Мета:** Спростити UI, прибрати зайву концепцію «системне поле», чітко розділити **тип даних** та **відображення**.

---

## Термінологія (не плутати)

| Термін | Поле в БД | Опис |
|--------|-----------|------|
| **Відображення** | `widgetType` | Як поле показується в UI: текстовий рядок, число, спадний список, складене поле, галерея, файли, дата тощо. |
| **Тип даних** | `dataType` | Що зберігається: текст, ціле число, дробне число, так/ні, дата. Визначає валідацію та формат. |

**Для складеного поля:** тип даних не має сенсу — поле містить підполя, кожне з яких має свій тип даних. Відображення = «Складене поле».

---

## Фаза 1: Прибрати колонку «Системне» з таблиці

**Файл:** `field-definitions-management.tsx`

| Дія | Деталі |
|-----|--------|
| Видалити таблицю | `TableHead` з текстом «Системне» |
| Видалити комірку | `TableCell` з `fd.isSystem ? Badge : "--"` |
| Оновити colSpan | `plain colSpan={6}` → `colSpan={5}` для порожнього рядка |

**Ризики:** Немає. Колонка не використовується.

---

## Фаза 2: Прибрати isSystem з форми редагування

**Файл:** `field-definitions-management.tsx`

| Дія | Деталі |
|-----|--------|
| Badge «Системне» | Видалити `{!isCreate && isSystem && (...)}` з SheetHeader |
| Код (readonly) | Прибрати `disabled={... \|\| (!isCreate && isSystem)}`, `readOnly={!isCreate && isSystem}` |
| Відображення | Прибрати гілку `!isCreate && isSystem ? Input : Select` — завжди Select |
| Тип даних | Прибрати гілку `!isCreate && isSystem ? Input : Select` — завжди Select (або приховано для composite) |
| Блок «Системна колонка» | Видалити `{!isCreate && isSystem && selectedField?.systemColumn && (...)}` |

**Змінні:** Можна залишити `isSystem` з API для сумісності, але не використовувати в UI.

---

## Фаза 3: Тип даних для composite — показувати «—»

**Файл:** `field-definitions-management.tsx`

| Місце | Було | Стане |
|-------|------|-------|
| Таблиця | `DATA_TYPE_LABELS[fd.dataType] ?? fd.dataType` | `fd.widgetType === "composite" ? "—" : (DATA_TYPE_LABELS[fd.dataType] ?? fd.dataType)` |
| Форма | `needsDataType` вже ховає блок для composite | Перевірити: для composite не показувати «Тип даних» |
| Збереження | При створенні composite — `dataType` може бути `string` (default) | При збереженні composite передавати `dataType: "string"` (або null для API) — залежить від схеми |

**Примітка:** `needsDataType = dataTypeOptions.length > 0`. Для composite `getDataTypesForWidget("composite")` повертає `[]`, тому блок «Тип даних» вже не показується. Потрібно лише виправити таблицю.

| Елемент | Покращення |

| Таблиця | Заголовки: «Тип даних» та «Відображення» — залишити як є |
